# CardinalSpace - Demo Script

- **Audience:** Stakeholders, investors, university administration

2. Show portal selection interface

## 🎭 Scene 2: User Registration & Onboarding (3 minutes)

**Presenter:** "Let me walk you through the user journey, starting with how simple it is to get started with CardinalSpace."

**Script & Actions:**

1. **Registration Process**
   - Click "User Portal" → "Sign Up"
   - Fill in registration form (use demo credentials)
   - Show email verification process
   - **Key Feature:** "Notice how quick and painless this is - under 60 seconds!"

2. **Profile Setup**
   - Complete profile information
   - **Important:** Show how name updates in header immediately
   - Add vehicle information
   - **Key Feature:** "Multiple vehicle support for families"

3. **Dashboard Overview**
   - Show personalized welcome message
   - Display quick stats and recent activity
   - **Key Feature:** "Everything you need at a glance"

## 🎭 Scene 3: Real-Time Parking Map (5 minutes)

**Presenter:** "This is where CardinalSpace truly shines - our real-time parking map that shows exactly what's available right now."

**Script & Actions:**

1. **Map Introduction**
   - Navigate to "Real-time parking map"
   - Show lot selection (A, B, C, D)
   - **Key Feature:** "Live data updates every 10 seconds"

2. **Advanced Filtering**
   - **First:** Show all available spaces (highlight the quantity)
   - **Filter by EV Charging:** "Look how easy it is to find EV spots"
   - **Filter by Accessibility:** "Perfect for users with special needs"
   - **Filter by Covered:** "Weather protection options"
   - **Clear Filters:** "And back to full view with one click"

3. **Space Selection**
   - Click on an available space
   - Show space details popup
   - **Key Feature:** "Each space has detailed information"

4. **Visual Indicators**
   - Point out the color-coded legend
   - Show the special feature indicators (♿, ⚡, 🏢)
   - **Key Feature:** "Intuitive visual design"

5. **Confirmation**
   - Show reservation confirmation

## 🎯 Demo Tips & Best Practices

### **Before the Demo**
- **Test All Features:** Ensure everything works smoothly
- **Prepare Test Data:** Have realistic parking scenarios ready
- **Check Internet Connection:** Ensure stable connection
- **Clear Browser Cache:** Fresh start for smooth performance
- **Have Backup Plan:** Screenshots/video if live demo fails

### **During the Demo**
- **Speak Confidently:** You know this product inside out
- **Engage Audience:** Ask questions, encourage participation
- **Focus on Benefits:** Not just features, but what they solve
- **Tell Stories:** Use real-world scenarios
- **Handle Questions:** Welcome interruptions, show expertise

### **Technical Demo Checklist**
- [ ] Landing page loads correctly
- [ ] Registration flow works
- [ ] Profile update reflects immediately
- [ ] Real-time map shows live data
- [ ] All filters work properly
- [ ] Reservation process completes
- [ ] Payment processing (demo mode)
- [ ] QR code generation
- [ ] Admin portal access
- [ ] All animations are smooth
- [ ] Web responsiveness across desktop browsers

### **Common Questions & Answers**

**Q: How do you handle real-time updates?**
A: "We use WebSocket connections to push updates every 10 seconds, ensuring users always see current availability."

**Q: What about security and privacy?**
A: "We use industry-standard JWT authentication, encrypt all sensitive data, and comply with privacy regulations."

**Q: How does pricing work?**
A: "We offer flexible pricing based on institution size, with options for per-user fees or revenue sharing models."

**Q: Can this integrate with existing systems?**
A: "Yes, we provide APIs for integration with student information systems, payment processors, and campus security systems."

**Q: What's the implementation timeline?**
A: "Typical implementation takes 4-6 weeks, including data migration, user training, and go-live support."
